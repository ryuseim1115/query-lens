# QueryLens

SQLを『読む』から『視る』へ。複雑なクエリの構造を解き明かす解析アシスタント

---

## 誰のどんな課題を解決するのか？

ターゲット  :複雑なテーブル構造に触れ、SQLの読解、作成に苦戦している若手の指導員  
解決する課題:若手エンジニアにとって、複数のテーブルが絡み合うSQLは理解困難である。指導員がコード解説に忙殺され、本来の業務工数を圧迫している。

---

## なぜそれを解決したいのか？

後輩指導の際に、口頭やテキストの説明では、お互いの脳内にある「データのつながり」を同期させるのに多大な時間がかかった。  
手元のSQLを一瞬で図解できるツールがあれば、「この図のここが、君のイメージとズレている」と、認識をすり合わせながら対話することができて、指導員は本来の業務に着手することができると思っているため。

---

## どうやって解決するのか？

複雑なクエリを分割し、ステップごとの中間テーブルをプレビュー表示する機能を導入する。  
指導員と若手が同じ中間結果を視覚的に確認しながら対話することで、ブラックボックス化しがちな結合プロセスを透明化する。  
これにより、概念の認識齟齬を未然に防ぎ、効率的な技術継承を実現する。
